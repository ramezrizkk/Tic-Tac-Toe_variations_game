◆ setup_players()

Player< char > ** WordGame_UI::setup_players ( )
overridevirtual

Custom setup for players, uses 'L1' and 'L2' as default symbols.

Custom setup for players. The symbols are not 'X' or 'O', but any letter. However, we'll use a placeholder symbol 'L' followed by the player number to distinguish them in the Player object, but the actual move symbol will be the letter the player enters. The base Player class requires a T symbol, so we'll pass '1' and '2' as T=char.

NOTE: The original X-O setup used static_cast<T>('X') and 'O', which is reasonable. For this game, the actual move symbol is determined during get_move. We'll stick to 'X' and 'O' for the Player object's internal symbol, but prompt for the actual letter in get_move.
